## Overview

Send emails in your subscribers' preferred languages. Create multiple language versions of your email, and Migma automatically sends the correct version to each subscriber based on their language preference.

## How It Works

<Steps>
  <Step title="Create Email in Multiple Languages">
    Generate your email with language versions:

    ```text
    Create a product launch email.

    Generate versions in:
    - English
    - Spanish
    - French

    Product: New Widget Pro
    Key benefits: AI-powered, 10x faster, easy to use
    CTA: Start Free Trial
    ```

    Migma creates separate versions for each language
  </Step>

  <Step title="Set Subscriber Languages">
    Ensure your subscribers have language preferences set:

    **Option 1: Manual Entry**
    - Go to Audience â†’ Manage Contacts
    - Edit subscriber
    - Set "Language" field (e.g., `en`, `es`, `fr`)

    **Option 2: CSV Import**
    - Include `language` column in CSV
    - Values: `en`, `es`, `fr`, `de`, etc. (ISO 639-1 codes)

    **Option 3: Auto-Detect from Country**
    - Set subscriber's `country` field
    - Migma automatically detects language (e.g., Mexico â†’ Spanish)
  </Step>

  <Step title="Send to Audience">
    When you send, Migma automatically:
    1. Checks each subscriber's language preference
    2. Sends the matching language version
    3. Falls back to default language if no match
  </Step>
</Steps>

## Language Detection Priority

**Migma determines subscriber language in this order:**

1. **Subscriber's `language` field** (if set explicitly)
   - Example: `language: "es"` â†’ Spanish
   - Highest priority - always used if present

2. **Country-to-language mapping** (if `language` not set)
   - Example: `country: "MX"` â†’ Spanish
   - Automatic detection based on country

3. **Fallback to default** (if neither set)
   - Uses your email's primary language (usually English)
   - Ensures everyone receives some version

**Example scenarios:**

| Language Field | Country Field | Result |
|----------------|---------------|--------|
| `es` | `US` | Spanish (explicit preference) |
| *(not set)* | `MX` | Spanish (detected from Mexico) |
| *(not set)* | *(not set)* | English (fallback default) |
| `en` | `MX` | English (explicit overrides country) |

## Sending Localized Emails

### Send with Language Matching

<Steps>
  <Step title="Create Multi-Language Email">
    Generate or have email versions ready in multiple languages
  </Step>

  <Step title="Click Send">
    Navigate to **Audience â†’ Send Emails**
  </Step>

  <Step title="Select Audience">
    Choose who receives the email:
    - All subscribers
    - Specific tags
    - CSV list
  </Step>

  <Step title="Migma Matches Languages">
    **Automatic matching happens:**

    For each subscriber, Migma:
    1. Checks their `language` field
    2. Falls back to country-detected language
    3. Selects matching email version

  </Step>

  <Step title="Send">
    Click "Send" - emails go out in correct languages
  </Step>
</Steps>

## Best Practices

<Check>**Always provide a default language** - Usually English as fallback</Check>

<Check>**Use native speakers for translations** - Don't rely solely on AI translation for important campaigns</Check>

<Check>**Keep content length similar** - Some languages are more verbose (German, French) than others</Check>

<Check>**Test all language versions** - Preview each version before sending</Check>

<Check>**Cultural adaptation** - Not just translation - adapt imagery, examples, humor for each culture</Check>

<Check>**Set languages early** - Import or set subscriber languages before campaigns</Check>

<Check>**Update language preferences** - Let subscribers choose their language in preference center</Check>

## Cultural Considerations

### More Than Translation

**Good localization includes:**

**Content adaptation:**
- Date formats (MM/DD/YYYY vs DD/MM/YYYY)
- Currency symbols ($, â‚¬, Â£, Â¥)
- Time zones and scheduling
- Holidays and seasonal references
- Cultural references and idioms

**Visual adaptation:**
- Images appropriate for each market
- Color meanings (red = luck in China, danger in West)
- Reading direction (RTL for Arabic/Hebrew)
- Product availability by region

**Tone and formality:**
- German/Japanese: More formal
- Spanish/Italian: More warm and personal
- French: Balance of formal and friendly
- English: Casual and direct

### Examples

**Bad (literal translation):**
```
English: "Black Friday Sale - Don't miss out!"
Spanish: "Venta de Viernes Negro - Â¡No te pierdas!"
```

**Good (culturally adapted):**
```
English: "Black Friday Sale - Don't miss out!"
Spanish: "Ofertas de Fin de Semana - Â¡Aprovecha ahora!" (Weekend Deals)
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Wrong language sent" icon="language">
    **If subscribers receive wrong language:**

    **Check:**
    - Subscriber's `language` field is correct
    - Country code is accurate (affects auto-detection)
    - Language code uses correct format (`es` not `esp`)
    - Email versions actually exist for that language

    **Fix:**
    - Update subscriber language field
    - Re-import with correct language codes
  </Accordion>

  <Accordion title="No language match" icon="circle-xmark">
    **If subscribers don't match any language:**

    **Check:**
    - Email has versions in required languages
    - Subscriber has `language` or `country` set
    - Language codes match exactly

    **What happens:**
    - Migma sends default/primary language version
    - No emails are lost - fallback ensures delivery
  </Accordion>

  <Accordion title="Can't set subscriber language" icon="user-xmark">
    **If language field not available:**

    **Solutions:**
    - Update via CSV with `language` column
    - Use API to set language programmatically
    - Set country field instead (auto-detects language)
    - Contact support if field is missing
  </Accordion>

  <Accordion title="Translation quality issues" icon="spell-check">
    **If AI translations aren't good enough:**

    **Solutions:**
    - Manually edit each language version
    - Use professional translation service
    - Have native speakers review
    - Provide glossary/brand terms to AI
    - Use "Generate in [language]" separately for each
  </Accordion>
</AccordionGroup>

## API Integration

**Set subscriber language via API:**

```javascript
// Update subscriber language
await fetch('https://api.migma.ai/v1/subscribers/{id}', {
  method: 'PATCH',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    language: 'es', // Spanish
    country: 'MX'   // Mexico (optional, for additional context)
  })
});
```

**Create multi-language email via API:**

```javascript
// Generate email in multiple languages
await fetch('https://api.migma.ai/v1/emails/generate', {
  method: 'POST',
  body: JSON.stringify({
    prompt: "Create welcome email",
    languages: ['en', 'es', 'fr'],
    projectId: "your_project_id"
  })
});
```
