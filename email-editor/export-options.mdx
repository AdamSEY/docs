## ESP-Specific Variables

<iframe width="100%" height="300" src="https://youtube.com/updatelater" title="ESP Variables in Migma" frameborder="0" allowfullscreen></iframe>

### How Migma Handles Dynamic Variables

Migma **preserves** ESP variables during export. You can use any template language:

<AccordionGroup>
  <Accordion title="HubSpot (HubL)" icon="hubspot">
    **Syntax:** `{{ variable_name }}`

    **Example in Migma:**
    ```html
    Hello {{ contact.firstname }}!

    Your order #{{ content.order_number }} has shipped.

    {% if contact.vip_status %}
    Thanks for being a VIP member!
    {% endif %}
    ```

    **Migma preserves:**
    - Variables: `{{ contact.* }}`
    - Conditionals: `{% if %}...{% endif %}`
    - Loops: `{% for %}...{% endfor %}`
    - Filters: `{{ variable|upper }}`

    **Export:** Variables remain intact in HTML
  </Accordion>

  <Accordion title="Mailchimp (Merge Tags)" icon="mailchimp">
    **Syntax:** `*|VARIABLE|*`

    **Example in Migma:**
    ```html
    Hi *|FNAME|*,

    This email was sent to *|EMAIL|*.

    *|IF:INTERESTED_IN_CATEGORY=technology|*
    Check out our latest tech products!
    *|END:IF|*

    *|HTML:CUSTOM_CONTENT|*
    ```

    **Migma preserves:**
    - Merge tags: `*|FNAME|*`, `*|LNAME|*`
    - Conditional content: `*|IF:|*...*|END:IF|*`
    - Custom HTML: `*|HTML:...|*`
  </Accordion>

  <Accordion title="Klaviyo (Liquid)" icon="database">
    **Syntax:** `{{ variable }}` and `{% tag %}`

    **Example in Migma:**
    ```liquid
    Hello {{ person.first_name | default: 'there' }}!

    {% if event.ProductName %}
    You viewed: {{ event.ProductName }}
    Price: {{ event.ProductPrice | money }}
    {% endif %}

    {% for item in event.Items %}
    - {{ item.ProductName }}: ${{ item.Price }}
    {% endfor %}
    ```

    **Migma preserves:**
    - Variables with filters
    - Conditionals
    - Loops
    - Custom properties
  </Accordion>

  <Accordion title="SendGrid (Handlebars)" icon="paper-plane">
    **Syntax:** `{{variable}}` and `{{#if}}...{{/if}}`

    **Example in Migma:**
    ```handlebars
    Hi {{firstName}},

    {{#if isPremium}}
    Premium features are now available!
    {{else}}
    Upgrade to unlock premium features.
    {{/if}}

    {{#each products}}
    - {{this.name}}: ${{this.price}}
    {{/each}}
    ```

    **Migma preserves:**
    - Variables
    - Conditionals
    - Loops
    - Helpers
  </Accordion>

  <Accordion title="Amazon SES (Mustache)" icon="aws">
    **Syntax:** `{{variable}}`

    **Example in Migma:**
    ```mustache
    Hello {{name}},

    Your subscription expires on {{expirationDate}}.

    {{#items}}
    - {{productName}}: {{price}}
    {{/items}}
    ```

    **Migma preserves:**
    - Simple variables
    - Sections (loops)
    - Inverted sections
  </Accordion>

  <Accordion title="Custom Variables" icon="code">
    **Any syntax supported!**

    Tell Migma which variables to preserve:

    ```text
    Create welcome email with these variables:
    - %%FIRSTNAME%%
    - %%SIGNUP_DATE%%
    - %%PLAN_NAME%%

    Use my ESP's %% syntax
    ```

    Migma recognizes and preserves your custom format
  </Accordion>
</AccordionGroup>

## How to Use ESP Variables

### During Email Creation

**In your prompts:**
```text
Create order confirmation email

Include:
- Customer name: {{ customer.name }}
- Order number: {{ order.number }}
- Order total: {{ order.total | currency }}
- Order items loop: {% for item in order.items %}

Use HubSpot HubL syntax
```

**Migma will:**
1. Generate email structure
2. Insert variables in correct locations
3. Preserve exact variable syntax
4. Export with variables intact

### In Visual Editor

**Add variables manually:**

1. Click text element
2. Type variable syntax directly: `Hello {{ firstname }}!`
3. Variables highlighted in editor
4. Preserved on export

**Or use variable inserter:**
1. Click "Insert Variable" button
2. Select ESP type (HubSpot, Mailchimp, etc.)
3. Choose variable from list
4. Inserted with correct syntax

## Email Client Compatibility

### How Migma Ensures Compatibility

<AccordionGroup>
  <Accordion title="Inline CSS" icon="paintbrush">
    **All styles inlined** for email clients that strip `<style>` tags

    **Before:**
    ```html
    <style>.button { background: blue; }</style>
    <a class="button">Click</a>
    ```

    **After export:**
    ```html
    <a style="background: blue; padding: 12px 24px; border-radius: 4px;">Click</a>
    ```
  </Accordion>

  <Accordion title="Table-Based Layouts" icon="table-columns">
    **Outlook compatibility** requires table layouts

    Migma uses:
    ```html
    <table role="presentation" cellpadding="0" cellspacing="0">
      <tr>
        <td><!-- Content --></td>
      </tr>
    </table>
    ```

    **Why:** Outlook uses Word rendering engine (doesn't support modern CSS)
  </Accordion>

  <Accordion title="Conditional Comments" icon="code">
    **Outlook-specific fixes:**

    ```html
    <!--[if mso]>
    <table><tr><td width="600">
    <![endif]-->

    <div style="max-width:600px;">
      <!-- Content -->
    </div>

    <!--[if mso]>
    </td></tr></table>
    <![endif]-->
    ```

    **Only Outlook sees** table wrapper
  </Accordion>

  <Accordion title="Web Fonts with Fallbacks" icon="font">
    **Google Fonts + fallbacks:**

    ```html
    <link href="https://fonts.googleapis.com/css?family=Open+Sans" rel="stylesheet">

    <p style="font-family: 'Open Sans', Arial, sans-serif;">
      Text content
    </p>
    ```

    **If font fails:** Falls back to Arial, then sans-serif
  </Accordion>

  <Accordion title="Dark Mode Support" icon="moon">
    **Automatic dark mode styles:**

    ```html
    <style>
    @media (prefers-color-scheme: dark) {
      .dark-text { color: #ffffff !important; }
      .dark-bg { background-color: #1a1a1a !important; }
    }
    </style>
    ```

    **Works in:** Apple Mail, Gmail, Outlook (iOS)
  </Accordion>

  <Accordion title="Responsive Design" icon="mobile">
    **Media queries for mobile:**

    ```html
    <style>
    @media only screen and (max-width: 600px) {
      .mobile-full-width { width: 100% !important; }
      .mobile-hide { display: none !important; }
      .mobile-stack { display: block !important; width: 100% !important; }
    }
    </style>
    ```

    **Columns stack** on mobile automatically
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Variables not working in ESP" icon="code">
    **Problem:** ESP doesn't recognize variables

    **Solutions:**
    - Verify correct syntax for your ESP
    - Check variable names match ESP fields
    - Ensure variables not accidentally encoded
    - Test with simple variable first
  </Accordion>

  <Accordion title="Layout broken in Outlook" icon="envelope-open-text">
    **Problem:** Email looks wrong in Outlook

    **Solutions:**
    - Re-export with "Outlook optimization" enabled
    - Ensure table-based layout
    - Check conditional comments intact
    - Avoid flexbox/grid CSS
  </Accordion>

  <Accordion title="Images not displaying" icon="image-slash">
    **Problem:** Images broken after upload

    **Solutions:**
    - Use absolute URLs (not relative)
    - Host images on CDN
    - Check image URLs accessible
    - Use Migma's CDN (automatic)
  </Accordion>

  <Accordion title="Dark mode issues" icon="moon">
    **Problem:** Unreadable in dark mode

    **Solutions:**
    - Enable "Dark mode styles" in export
    - Test with Email Preflight
    - Use dark-mode-safe colors
  </Accordion>
</AccordionGroup>
